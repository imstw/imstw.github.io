CRUD (SQL vs MongoDB)

圖片來源: https://university.mongodb.com/courses/M100/about

C – CREATE (insert)

db.collection.insertOne() =>括號內填入json格式document

db.collection.insertOne({ “field1” : value1 , ”field2” : value2, ”field3” : value3 })  
db.collection.insertMany(  
[{ “field1” : value1 , ”field2” : value2, ”field3” : value3 },  
{ “field1” : value4, ”field2” : value5 },  
{ “field1” : value4 , ”field2” : value5, ”field3” : value6 }])

—Unordered Insert無序插入 (當_id衝突時)—

db.test.insertMany([{_id:2,A:10},{_id:1,A:20},{_id:3,A:30}])

=>照順序插入，若id:1已經存在，會報錯，後續的id:3就不會被插入

db.test.insertMany([{_id:2,A:10},{_id:1,A:20},{_id:3,A:30}],{ordered:false})

=>即使前面有報錯，仍會跑完全部insert資料，id:3就會成功插入

R – READ (find)

findOne(filter,options)

find(filter,options)

db.employee.find({},{first_name:1,last_name:1,_id:0})

=>只傳回first_name、last_name

—複雜document filter—

db.employee.findOne({"contact.phone":1111})  
db.employee.find({hobby:["book","movie"]}) =>AND  
db.employee.find({hobby:{$in:["book","movie"]}}) =>OR(include)

—MongoDB Comparison Query Operators操作符—

db.employee.find({"salary":{"$gt":5000}})

=>找出所有salary大於5000的資料

—比較操作符—

$eq 等於 = $gt 大於 >
$gte 大於等於 >=
$in 包含其中任一個
$lt 小於 <
$lte 小於等於 <=
$ne 不等於 !=
$nin 不包含其中任一個

db.movie.find({imdb_score:{$gt:8}}).count()  
db.movie.find({imdb_score:{$lt:5}})

—邏輯操作符—

$and
$not
$nor or的相反
$or

db.movie.find({$and :[{imdb_score:{$gt:7}},{imdb_score:{$lt:8}}]}).count()

=>評分大於7且小於8的電影數

db.movie.find({$or :[{imdb_score:{$lt:7}},{imdb_score:{$gt:8}}]}).count()

=>評分小於7且大於8的電影數

db.movie.find({$nor :[{imdb_score:{$lt:7}},{imdb_score:{$gt:8}}]}).count()

=>評分不滿足小於7且大於8條件的電影數

—Element Query Operators元素操作符—

$exists 是否存在

db.movie.find({"imdb_score":{$exists: true, $in: [5,7]}})

=>找出imdb_score field存在且值為5或7的資料

$type 字段的數據類型是否滿足條件

db.movie.find({"director_name":{$type : "string"}})  
db.movie.find({"director_name":{$type : 2}})

=>找出"director_name"資料類型是"字串"的資料

—Array查詢操作符—

db.people.find({"hobby.type": "Movie"})  
db.people.find({hobby:{$elemMatch:{type: "Movie"}}})  
db.people.find({hobby:{$elemMatch:{type: "Movie", rating: 3}}})

—sort、limit、skip—

db.movie.find.sort({imdb_score:1})  =>由小到大  
db.movie.find.sort({imdb_score:-1})  =>由大到小  
db.movie.find.sort({imdb_score:1}).limit(3)  =>由小到大排序後，取前三筆  
db.movie.find.sort({imdb_score:1}).limit(3).skip(6)  =>由小到大排序後，從第七筆開始取前三筆

U – Update (update、replace)

updateOne(filter,update,options)

=>更新符合filter條件的第一筆document

updateMany(filter,update,options)

=>更新符合filter條件的所有document

update(filter,update,options)

=>覆蓋符合filter條件的所有document(慎重使用)

replaceOne(filter,replace,options)

=>覆蓋符合filter條件的第一筆document

— updateOne、replaceOne—

updateOne: 可更新指定fields，可以添加新字段而不會丟失舊document中的字段
replaceOne: 替換整個document

例如:   { "_id" : ObjectId("0123456789abcdef01234567"), "my_test_key3" : 3333 }  
使用: replaceOne({"_id" : ObjectId("0123456789abcdef01234567")}, { "my_test_key4" : 4})  
結果=> { "_id" : ObjectId("0123456789abcdef01234567"), "my_test_key4" : 4 } 

使用: updateOne({"_id" : ObjectId("0123456789abcdef01234567")}, {$set: { "my_test_key4" : 4}})  
結果=>  {"_id" : ObjectId("0123456789abcdef01234567"),  
   "my_test_key3" : 3333.0, "my_test_key4" : 4.0}

—update操作符—

$set
db.employee.updateOne({"_id" :ObjectId("626102574e3428417697274e")},{$set:{"salary":7300}})

=> 更新_id為ObjectId(“626102574e3428417697274e”)的”salary”field value為7300

$inc (加)
db.products.updateOne({ sku: "abc123" }, { $inc: { quantity: -2, "metrics.orders": 1 } })

=> quantity值減2、metrics.orders值加1

Update with an Aggregation Pipeline:
db.collection.updateOne(  
   <filter>,  
   [  
      { $set: { status: "Modified", comments: [ "$misc1", "$misc2" ] } },  
   ]  
)
設定upsert
db.people.updateOne({name: "Max"},{$set:{age:30}},{upsert: true})

=>沒有匹配到的話就insert一筆新的

$unset刪除字段、$rename
db.people.updateMany({},{$rename:{phone: "phoneNum"}})  
db.people.updateMany({},{$unset:{phoneNum: 1}})

D – DELETE (delete)

db.collection.deleteOne(filter,options) =>帶入filter條件

db.collection.deleteOne({"a":1})

符合”a”:1的資料會刪除，有重複的資料符合filter條件，則只會刪除第一筆

db.collection. deleteMany({"a":1})

有重複的資料符合filter條件都會刪除

大括弧 { } 裡面是存放條件用的，如果不帶條件則是刪除第一筆 或是 全部刪除
刪除全部資料：db.collection.deleteMany()
刪除整個 Collection(包含 Index)：db.collection.drop()
